
# Design Principles

These principles guide the design of the football data platform.

The goal is to create a clear, data-driven and highly scannable interface for football fans.

---

## 1 Clarity over decoration

Information should always be easy to scan.

Football data such as scores, rankings and statistics must be immediately visible.

Avoid visual noise.

---

## 2 Data first

The interface prioritizes data visibility.

Examples:

- Match scores
- Player statistics
- League tables
- Market values

Visual elements support the data but never dominate it.

---

## 3 Consistent components

Use existing components whenever possible.

Examples:

- player-row
- team-row
- match-card
- group-table

Avoid creating new components unless necessary.

---

## 4 Reusable patterns

Page sections are built from reusable patterns.

Examples:

- match-list
- leaderboard
- stats-section
- video-carousel
- news-feed

Patterns ensure visual consistency across pages.

---

## 5 Token driven design

All design values must use tokens.

Examples:

- spacing
- colors
- typography
- radius
- shadow

Avoid hardcoded values.

---

## 6 Mobile-first thinking

Design always starts with mobile layouts.

Layouts scale up to tablet and desktop.

---

## 7 Fast scanning

Football fans scan information quickly.

Use:

- clear headings
- grouped data
- cards for separation
- consistent spacing
